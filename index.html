Part 5
I/O Redirection & Pipes
Every process has three standard streams. Redirection rewires them; pipes chain processes together.
stdin (0)
Keyboard input
stdout (1)
Normal output
stderr (2)
Error messages
Redirection operators
OperatorMeaningExample
>Redirect stdout (overwrite)ls > out.txt
>>Append stdoutecho hi >> out.txt
<Redirect stdin from filesort < data.txt
2>Redirect stderrcmd 2> err.log
&>Both stdout & stderrcmd &> all.log
2>&1Merge stderr into stdoutcmd > out.log 2>&1
Pipelines
who | sort > sorted_users.txt
ls -l | sort | head -5
date | cut -f1 -d" "
du | sort -nr
Pipes connect stdout of one command to stdin of the next.
/dev/null — the black hole
ls > /dev/null
cmd > /dev/null 2>&1

Anything sent to /dev/null is permanently discarded — useful for silencing unwanted output.

Part 9
Looping Structures
for, while, and until — each suits a different scenario. while loops while TRUE, until loops while FALSE.
for loop — list style
for num in 1 2 3 4 5; do
    echo $num
done

for i in {1..10}; do
    echo $i
done

for file in *.txt; do
    echo "Processing $file"
done
for loop — C style
for ((i=0; i<=5; i++)); do
    echo $i
done

# Nested loops
for ((i=1; i<=3; i++)); do
    for ((j=1; j<=3; j++)); do
        echo -n "$i×$j "
    done; echo
done
while loop
counter=1
while [ $counter -le 10 ]; do
    echo $counter
    ((counter++))
done

# Read file line by line
while read LINE; do
    echo $LINE
done < /etc/passwd
until + break/continue
until [ $counter -gt 10 ]; do
    echo $counter
    ((counter++))
done
for i in 1 2 3 4 5; do
    [ $i -eq 3 ] && break
    echo $i       # prints 1 2
done
Part 10
Functions
Functions group reusable commands, keep scripts modular, and accept arguments via $1, $2 etc.
Defining & calling
greet() {
    echo "Hello $1, welcome!"
    return 0
}

function add() {
    echo $(( $1 + $2 ))
}

greet "World"   # → Hello World, welcome!
add 5 3         # → 8

# Capture return data
result=$(add 5 3)
echo "Sum is $result"
Key rules
  • Functions must be defined before they are called
  • $1, $2, etc. inside a function refer to function arguments, not script args
  • return exits the function; exit exits the whole script
  • Return values are status codes (0–255) — use echo + $() to pass data
shift inside functions
process_all() {
    while [ $# -gt 0 ]; do
        echo "Arg: $1"
        shift
    done
}
process_all apple banana cherry

Part 12
AWK — Pattern Scanning & Processing
AWK (Aho, Weinberger, Kernighan) treats text as structured records and fields — ideal for data extraction and reports.
Syntax & built-in variables
awk '/pattern/ { action }' file
# No pattern → action runs on every line
# No action  → default action is print
VariableMeaning
$0Entire current line
$1, $2, ...Individual fields (whitespace-separated by default)
NFNumber of fields in current record
NRCurrent line number
FSField separator (change with -F)
$NFLast field
AWK examples
awk '{print}' file                   # print all lines
awk '{print $2, $5}' file            # print fields 2 and 5
awk '{print $NF}' file               # print last field
awk '/Nihal/ || /Nishadi/' file      # match patterns
awk '$1 > 200' file                  # numeric condition
awk '$4 ~ /Technology/' file         # field matches regex
awk -F: '{print $1}' /etc/passwd     # colon separator

# BEGIN and END
awk 'BEGIN {print "Name\tSalary"}
     {print $2, $5}
     END {print "Report done"}' emp.txt

# Count matching lines
awk 'BEGIN {c=0} $4~/Tech/{c++} END{print "Count:",c}' emp.txt
Part 13
SED — Stream Editor
SED edits files without opening them — perfect for find-and-replace, deletion, and transformation at scale.
Substitute command s///
CommandEffect
sed 's/unix/linux/' fileReplace first occurrence per line
sed 's/unix/linux/2' fileReplace second occurrence per line
sed 's/unix/linux/g' fileReplace ALL occurrences (global)
sed '3 s/unix/linux/' fileReplace only on line 3
sed '1,3 s/unix/linux/' fileReplace on lines 1–3 only
sed -n 's/unix/linux/p' filePrint only changed lines
sed -i 's/unix/linux/g' fileEdit in-place (modifies original!)
Delete commands
sed '5d' file         # delete line 5
sed '12,$d' file      # line 12 to end
sed '/abc/d' file     # lines matching abc
sed '/^$/d' file      # empty lines
Other commands
sed G file            # blank line after each
sed -n '5,10p' file   # print lines 5-10
sed '3i\New line' f   # insert before line 3
sed '3a\New line' f   # append after line 3
-i warning: sed -i modifies the original file permanently. Always back up first: cp file file.bak

Part 15
POSIX Threads (pthreads)
Threads are lightweight execution units inside a process — they share the same memory space and run concurrently.
Process vs Thread
FeatureProcessThread
MemoryOwn (separate)Shared
WeightHeavyLightweight
Context switchSlowFast
Creationfork()pthread_create()
pthread functions
FunctionPurpose
pthread_create()Create a new thread
pthread_join()Wait for thread to finish
pthread_exit()Terminate calling thread
pthread_mutex_lock()Lock a mutex
pthread_mutex_unlock()Unlock a mutex
Simple thread example
#include <stdio.h>
#include <pthread.h>

void* addition(void* arg) {
    printf("Sum = %d\n", 10+20);
    pthread_exit(NULL);
}

int main() {
    pthread_t t1;
    pthread_create(&t1, NULL, addition, NULL);
    pthread_join(t1, NULL);
    return 0;
}

// Compile:
gcc program.c -o program -pthread
Race condition & mutex fix
// WITHOUT mutex — RACE CONDITION!
int counter = 0;
void* inc(void* arg) {
    for(int i=0;i<100000;i++)
        counter++;  // unsafe!
}

// WITH mutex — correct
pthread_mutex_t m = PTHREAD_MUTEX_INITIALIZER;
void* inc_safe(void* arg) {
    for(int i=0;i<100000;i++) {
        pthread_mutex_lock(&m);
        counter++;
        pthread_mutex_unlock(&m);
    }
}
Without mutex, two threads can produce a result less than 200,000 due to race conditions.
